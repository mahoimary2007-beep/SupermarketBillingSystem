1. Project Description
The Supermarket Billing System is an automated, terminal-based Python application designed to streamline retail checkout workflows. It allows cashiers to efficiently process multi-item customer transactions, apply customized percentage discounts, and generate mathematically precise, itemized digital receipts using Leones (Le) currency.

2. How the Program Works
**Data Input:** The cashier inputs the product name, unit price, and quantity for each item purchased.
**Nested Loops:** A structured loop continuously updates line-item calculations until the session is closed.
**Discount Processing:** The system automatically calculates and deducts percentage discounts before final totals.
**Receipt Generation:** Breaks out of entry loops to display a structured transaction summary in Leones (Le).

 3. How to Run the Program
1. Ensure **Python 3.x** is installed on your operating system.
2. Clone or download this repository containing `Supermarket Billing System.py`.
3. Open the file directly inside **Visual Studio Code (VS Code)**.
4. Click the solid **Play Button (▶️)** in the top-right corner of the editor interface.
5. Control inputs and run transactions dynamically within the open **Terminal** window.

 4. Documentation: Problem Solved
Traditional, manual shop checkouts cause processing delays, structural accounting mistakes, and cash discrepancies. This computational framework secures complete mathematical accuracy for dynamic multi-item entries, structures discount allocations perfectly, and guarantees instant financial tracking for every customer transaction.

 5. Features
 **Dynamic Transaction Entry:** Processes multiple distinct line items iteratively per checkout session.
 **Automated Discounts:** Direct, reliable integration of custom percentage reductions to replace human calculation errors.
 **Local Currency Integration:** Core output interface formatted explicitly around Leones (Le).
 **Robust Loop Constraints:** Built using reliable logical control conditions to ensure smooth runtime data flow.

 6. Future Improvements
 **Graphical User Interface (GUI):** Migrating system architecture out of the terminal into a visual desktop layout using Tkinter.
 **Database Integration:** Linking system storage to a centralized SQL database for live inventory tracking.
 **Hardware Support:** Synchronizing scanning input parameters to read standard retail barcode hardware directly.

 7. Business and Societal Impact
As an accessible Digital Public Good, this system lowers technology costs for localized small-to-medium enterprises (MSMEs). Moving away from paperwork structures helps local shops shorten checkout line wait times, secure accurate daily balance tracking, and build transparent pricing trust within their communities.
